Lyocell — Load Testing, Minimal Setup

lyocell@local:~$ ./lyocell landscape

Run JavaScript, spawn thousands of virtual users, and export shareable HTML reports. Zero servers, zero boilerplate—just a fast CLI.

Install

macOS / Linux (Homebrew)

brew tap wilhg/lyocell
brew install lyocell

Linux (direct)

wget https://github.com/wilhg/lyocell/releases/latest/download/lyocell-linux-amd64
chmod +x lyocell-linux-amd64
sudo mv lyocell-linux-amd64 /usr/local/bin/lyocell

Windows (Scoop)

scoop bucket add lyocell https://github.com/wilhg/lyocell-scoop
scoop install lyocell

Run a Test

// test.js
import http from 'lyocell/http';
import { check, sleep } from 'lyocell';

export const options = {
  vus: 20,
  duration: '30s',
  thresholds: {
    http_req_duration: ['p(95)<500'],
    checks: ['rate>0.99'],
  },
  lyocell: {
    outputs: [{ type: 'html', target: 'report.html' }],
  },
};

export default function () {
  const res = http.get('https://httpbun.com/get');
  check(res, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}
lyocell test.js -o html=report.html
Get a console summary and an offline HTML report—perfect for complex load tests and sharing results.

Why Lyocell

  • Familiar JS Minimal learning curve for scripting.
  • Simple setup No extra runtimes or services to manage.
  • Fast start Instant boot; great for CI and local dev.
  • HTML reports Zero-dependency, shareable summaries.

Scenarios (CLI-friendly)

Ramping VUs

export const options = {
  scenarios: {
    ramp: {
      executor: 'ramping-vus',
      stages: [
        { duration: '30s', target: 50 },
        { duration: '2m',  target: 50 },
        { duration: '20s', target: 0 },
      ],
    },
  },
};

Constant Arrival Rate

export const options = {
  scenarios: {
    api: {
      executor: 'constant-arrival-rate',
      rate: 200,
      timeUnit: '1s',
      duration: '1m',
      preAllocatedVUs: 50,
      maxVUs: 200,
    },
  },
};

Outputs

  • Console Live summary as the test runs.
  • HTML `-o html=report.html` for offline, shareable reports.

Metrics & Checks

  • HTTP durations (p95/p99), iterations, checks, custom counters/trends.
  • Thresholds fail fast when SLOs are missed.
